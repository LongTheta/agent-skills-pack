# Repo Docs Writer — Examples

## Example 1: README Improvement

**Input:** "Improve our README; it's sparse"

**Output:** Structured README with: Project overview, Features, Install, Usage (with example), Configuration, Contributing link, License.

## Example 2: CONTRIBUTING

**Input:** "Write a CONTRIBUTING.md for our Python project"

**Output:** PR process, branch naming, coding style (black, ruff), test command, how to run locally.

## Example 3: Architecture Doc

**Input:** "Create an architecture overview for our microservices"

**Output:** High-level diagram (ASCII or Mermaid), component list, data flow, key decisions.
